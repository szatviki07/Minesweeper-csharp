\# Minesweeper – Multi-Framework C# Implementation



\## Overview

This project implements the classic Minesweeper game in C# using three different UI frameworks:



\- WinForms

\- WPF (MVVM architecture)

\- Avalonia (cross-platform, including Android \& Desktop)



\## Architecture

The application follows a layered design:



\- \*\*Model layer\*\* – game logic and rules

\- \*\*Persistence layer\*\* – file-based data handling

\- \*\*View / ViewModel layer\*\* – UI logic (MVVM in WPF \& Avalonia)

\- \*\*Unit Tests\*\* – model-level automated tests (MSTest)



Key concepts practiced:

\- Object-Oriented Programming (OOP)

\- Event-driven programming

\- MVVM pattern

\- Separation of concerns

\- Cross-platform UI development

\- Unit testing



\## Implementations

\### WinForms

\- Event-driven UI

\- Direct interaction with game model



\### WPF

\- MVVM pattern

\- Data binding

\- DelegateCommand usage



\### Avalonia

\- Cross-platform UI framework

\- Desktop and Android targets

\- MVVM-based structure
